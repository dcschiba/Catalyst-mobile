WRAP-TestSuite（開発者向け）

テストコードの書き方

テストコードはNode.jsのライブラリであるselenium-webdriverを使用します。

ブラウザ間の差異について

ブラウザを使ったE2Eテストでは、ブラウザごとの挙動の違いに影響を受けやすく、また、Selenium WebDriverの仕様のベースとなっているWebDriverはブラウザによって対応状況に違いがあります。
そのため、Chromeではうまく動いたテストコードがIEでは動かない、など、ブラウザ間でテストコードの実行結果に差異が生じる場合がありますので、適宜、ブラウザに合わせたテストコードのチューニングが必要となります。

テストコードの準備

WRAPの提供するテンプレートに同梱されているindex.jsを使用してください。
ローカル環境でテストコードを実行する際は、テストコード内のhubServer変数に設定されているアドレスを「localhost」に変更します。 

const hubServer = 'http://localhost:4444/wd/hub'; // Hubサーバ
...

const driver = new webdriver.Builder().usingServer(hubServer).withCapabilities(capabilities).build();

テストコードの単位

テストコードはselenium-webdriverが提供するAPIを使用しますが、内部ではJavaScriptのテスティングフレームワークであるMochaが使われていますので、書き方はMochaと同様です。
describeで画面や機能単位でテストを分割し、it内に具体的なテストケースを記載します。

t.describe('Car-GPS', function () {
    // 前処理
    t.it('Car-GPS_check_CarGPS_checkbox', (done) => {
        // テスト処理
    });
});

テストケース名のルール

itに設定するテストケース名は「機能名（画面名）テスト内容」を設定してください。
テストケース名がそのまま取得するキャプチャのファイル名になりますので、スペースは使用せず、（アンダーバー）で文字間は繋ぐようにしてください。 

t.it('Radar_validtime_change')

要素の取得

HTMLの要素の取得はselenium-webdriverが提供するfindElementとByを使用します。

  • 例：idがmapのdivタグ

    driver.findElement(By.id('("map")'))

  • 例：altにweathernewsが指定されているimgタグ

    driver.findElement(By.xpath('(//img[alt="weathernews"]'))

  • 例：深い階層にいるdivタグ

    driver.findElement(By.xpath('//div[@id="root"]/div/div[2]/div/div/div[2]/div'))

各APIの詳細はリファレンスサイトをご参照ください。

また、xpathでの指定は様々な記載方法がありますので、MSDNの解説サイトなどをご参照ください。

要素の操作

要素を取得した後は、要素に対してclickなどの様々な操作を行うことができます。

  • 例：要素をクリック 
    driver.findElement(By.xpath('(//img[alt="weathernews"]')).click();

詳細はリファレンスサイトをご参照ください。

ブラウザへ直接JavaScriptの操作を行う場合

ブラウザの操作はselenium-webdriverが提供するAPIである程度行うことができますが、より細かい操作など、ブラウザへ直接命令を送りたい場合はexecuteScriptを使用します。

  • 例：ブラウザ上でalertを実行 
    driver.executeScript('alert("hello")');

次の処理の実行タイミング

テストコードの実行にはブラウザの操作が伴うため、実行のタイミングに気を付ける必要があります。
例：ボタンAを押した後、画像Bが表示されるのを待ち、ボタンCを押す、など

そのため、必要に応じて、次の処理が実行可能になるまでテストコード上で処理を待機させる必要があります。

  • 例：次の処理まで2秒待機

    driver.sleep(2000);

  • 例：classにmainを持った要素が描画されるまで待機

driver.wait(webdriver.until.elementLocated(By.className('main')), 2000);

ブラウザや端末ごとに描画の速度にも違いが出てくるため、なるべく要素の有無などで次の処理の実行可否を判断するようにしてください。

非同期処理の対応

ブラウザの操作を伴うAPIの大半は非同期処理となっています。
コールバックによってテストコードの複雑化を防ぐためにPromiseを使って非同期処理の制御をしてください。

  • 例：URLにアクセス→描画待機→チェックボックスクリック→2秒待機→キャプチャ取得
      t.it('click_checkBox', (done) => {
      driver.get('http://...')
          .then(() => driver.wait(webdriver.until.elementLocated(By.className('main')), 2000)) // mainの描画を待つ
          .then(() => driver.findElement(By.xpath('(//div[contains(@style, "cursor: pointer")])[2]')).click()) // チェックボックスをクリック
          .then(() => driver.sleep(2000)) // 2秒待機
          .then(() => {
              driver.saveScreenshot('./capture.png'); // キャプチャを取得
              done();
          });
  });

マジックナンバーを避ける

座標の指定やselectボックスの指定などに使用する値は、テストコードの保守性を高めるためになるべく変数化するようにしてください。

テスト結果の判定

テスト結果の判定はNode.jsの提供するassertを使用します。

  • 例：result.equalityが0であるかの判定 
    assert.equal(result.equality, 0, '比較結果に差分があります。');

コメントの記入

テストコード内のコメントからテスト仕様書を自動的に生成するようにしていますので、ブラウザの操作を伴う処理ごとに@testをつけたコメントを記載してください。

driver.findElement(By.xpath('(//img[alt="weathernews"]')).click(); // @test ロゴ画像をクリック
...
driver.findElement(By.xpath('//body/div/div/div[2]/div/div/span/div/div/div/div')).click()) // @test チェックボックスをクリック
...
driver.saveScreenshot('./capture.png'); // @test キャプチャを取得

その他、テストコードの実装にあたっての注意事項等は、Tips（テストコード作成）をご参照ください。
