キャプチャ処理

キャプチャの取得

現在ブラウザに表示されている画面を取得し、サーバ内に画像保存します。

selenium webdriverが提供するtakeScreenshotとnode.jsのfsを使用しています。

キャプチャの保存先

jenkinsのpluginと連携させるため、キャプチャは以下の保存先、ファイル名で保存してください。
カレントディレクトリ/describeに設定したテスト名/itに設定したテストケース名.png
カレントディレクトリはprocess.cwd()、describeに設定したテスト名はdescribeの関数内でthis.title、テストケース名はitの関数内でthis.test.titleでそれぞれ取得することができます。
キャプチャの取得はテンプレートで用意しているsaveScreenshotを呼び出してください。 

t.describe('Car-GPS', function () {
    const currentDir = process.cwd();
    const screenshotDir = path.join(currentDir, this.title); // キャプチャ保存先ディレクトリ
    mkdir(screenshotDir); // ディレクトリ作成
    t.it('Car-GPS_check_CarGPS_checkbox', (done) => {
        const screenShotFile = path.join(screenshotDir, `${this.test.title}.png`); // キャプチャ保存先ディレクトリとファイル名指定
        driver.saveScreenshot(screenShotFile);
    });
});

saveScreenShotは非同期処理のためpromiseを使用して処理を制御してください。

キャプチャの抜き出し

取得したキャプチャからmap部分のみを抜き出したい場合など、画像を加工することができます。

ImageMagickとImageMagickをNode.js上から呼び出すgmを使用しています。

  • 範囲を指定して抜き出し
    対象の位置や高さ・幅を指定して画像を抜き出すことができます。
    テンプレートで用意しているcropImageByRectを呼び出してください。
    driverから呼び出しが可能です。 
const rect = {};
rect.height = 30; // 高さ
rect.width = 40; // 幅
rect.left = 15; // 画面左からの絶対位置
rect.top = 15;  // 画面上からの絶対位置
const beforeImage = './before.png';
const afterImage = './after.png';

driver.cropImageByRect(beforeImage, afterImage, rect);

非同期処理のためpromiseを使用して処理を制御してください。

  • elementを指定して抜き出し
    指定したdivタグなど、elementを指定して画像を抜き出すことができます。
    テンプレートで用意しているcropImageByElementを呼び出してください。
    driverから呼び出しが可能です。 
driver.cropImageByElement('./before.png', './after.png', driver.findElement(By.id('map')));

非同期処理のためpromiseを使用して処理を制御してください。

キャプチャの比較

取得したキャプチャ（テスト結果）の正誤判定にキャプチャの比較をすることができます。
こちらも同様にImageMagickとImageMagickをNode.js上から呼び出すgmを使用しています。

テンプレートで用意しているcompareImageを呼び出すことで比較が可能です。 

const image1 = './image1.png'; // 比較画像1
const image2 = './image2.png'; // 比較画像2
const options = {
    file: 'image_diff.png', // 比較結果画像
};
driver.compareImage(image1, image2, options);

比較元画像はサーバ上に配置した画像を利用するため、以下のディレクトリ、ファイル名で統一してください。
カレントディレクトリ/master/describeに設定したテスト名/itに設定したテストケース名.png

キャプチャの取得から比較までのテストコードは以下の形になります。 

t.describe('Car-GPS', function () {
    const currentDir = process.cwd();
    const screenshotDir = path.join(currentDir, this.title);
    mkdir(screenshotDir);
    const masterScreenshotDir = path.join(currentDir, 'master', this.title); // 比較元画像の保存先
    t.it('Car-GPS_check_CarGPS_checkbox', (done) => {
        const screenShotFile = path.join(screenshotDir, `${this.test.title}.png`);
        const masterScreenShotFile = path.join(masterScreenshotDir, `${this.test.title}.png`); // 比較元画像のファイル名
        const diffScreenShotFile = path.join(screenshotDir, `${title}_diff.png`); // 比較結果のファイル名
        driver.saveScreenshot(screenShotFile)
            .then(() => {
                const options = {
                    file: diffScreenShotFile,
                };
                return driver.compareImage(masterScreenShotFile, screenShotFile, options);
            }).then((result) => {
                assert.equal(result.equality, 0, `比較結果に差異があります。【${testTitle}_diff.png】を確認してください。`);
            })
    });
});

非同期処理のためpromiseを使用して処理を制御してください。
比較結果に差異がない場合はcompareImageが返却するequalityの値が0になりますので、そちらの値でテスト結果を判定します。
比較画像間の高さ、幅のサイズに違いがあるとエラーになりますのでご注意ください。

result.equalityの値が0（テスト結果OK）の際は、比較結果の画像ファイルは削除してください。 

if (result.equality === 0) {
    removeFile(diffScreenShotFile);
}
